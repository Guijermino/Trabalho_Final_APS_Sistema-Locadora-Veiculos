# Sistema de Locadora de Veículos

## Descrição do Sistema

O sistema tem como objetivo gerenciar uma locadora de veículos, permitindo o cadastro de clientes, veículos e funcionários, além do controle de locações, devoluções, pagamentos e emissão de relatórios.

---

# Requisitos Funcionais (RF)

## RF01 - Cadastrar Cliente

O sistema deve permitir o cadastro de clientes.

## RF02 - Consultar Cliente

O sistema deve permitir consultar clientes cadastrados.

## RF03 - Alterar Cliente

O sistema deve permitir alterar dados dos clientes.

## RF04 - Excluir Cliente

O sistema deve permitir excluir clientes sem locações ativas.

## RF05 - Cadastrar Veículo

O sistema deve permitir cadastrar veículos.

## RF06 - Consultar Veículo

O sistema deve permitir consultar veículos cadastrados.

## RF07 - Alterar Veículo

O sistema deve permitir alterar dados dos veículos.

## RF08 - Excluir Veículo

O sistema deve permitir excluir veículos sem locações vinculadas.

## RF09 - Verificar Disponibilidade

O sistema deve permitir verificar a disponibilidade de veículos.

## RF10 - Registrar Locação

O sistema deve permitir registrar uma locação.

## RF11 - Registrar Devolução

O sistema deve permitir registrar a devolução de veículos.

## RF12 - Calcular Valor da Locação

O sistema deve calcular automaticamente o valor total da locação.

## RF13 - Registrar Pagamento

O sistema deve permitir registrar pagamentos.

## RF14 - Emitir Relatório de Veículos

O sistema deve gerar relatórios dos veículos cadastrados.

## RF15 - Emitir Relatório de Locações

O sistema deve gerar relatórios das locações realizadas.

## RF16 - Cadastrar Funcionário

O sistema deve permitir cadastrar funcionários.

## RF17 - Realizar Login

O sistema deve permitir autenticação por login e senha.

---

# Requisitos Não Funcionais (RNF)

## RNF01 - Interface

O sistema deve possuir interface intuitiva e de fácil utilização.

## RNF02 - Desempenho

As consultas devem ser respondidas em até 3 segundos.

## RNF03 - Segurança

O acesso deve ser realizado por login e senha.

## RNF04 - Persistência

Os dados devem ser armazenados em banco de dados.

## RNF05 - Confiabilidade

O sistema deve validar os dados informados.

## RNF06 - Compatibilidade

O sistema será desenvolvido em Java utilizando Programação Orientada a Objetos.

## RNF07 - Manutenibilidade

O código deve seguir boas práticas de orientação a objetos.

## RNF08 - Versionamento

O projeto deve ser mantido em um repositório GitHub.

## RNF09 - Documentação

Toda a documentação deve permanecer atualizada no repositório.
